# 2026-04-26 - v0.5.16 Bridge Oracle P2P Redeploy

## Objective

Close bridge and oracle production-readiness gaps on testnet, fix external P2P self-endpoint learning without legacy compatibility paths, cut a signed release, and clean-slate redeploy all three VPSes from the exact release artifact.

## Code Changes

- `p2p/src/network.rs` now derives an advertised address from `P2PConfig.external_addr` and uses it for gossip and outgoing P2P messages.
- `p2p/src/peer.rs` now stores an optional external address in `PeerManager`, rejects self endpoints before outbound dial, removes self endpoints from `PeerStore`, and removes stale entries when a same-node identity is detected.
- `p2p/src/gossip.rs` now uses `PeerManager::is_self_endpoint` during reconnect and peer-info discovery, so a VPS bound to `0.0.0.0:7001` will not re-add its own public `IP:7001`.
- `scripts/clean-slate-redeploy.sh` now sends explicit `Accept` and `User-Agent` headers in the Python protocol bootstrap verifier to avoid false Cloudflare 403s seen with bare `urllib`.
- Version bumped to `0.5.16`.
