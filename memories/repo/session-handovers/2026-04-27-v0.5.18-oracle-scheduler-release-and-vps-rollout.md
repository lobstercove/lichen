# 2026-04-27 - v0.5.18 Oracle Scheduler Release and VPS Rollout

## Objective

Cut the corrected release after the `v0.5.17` rollout exposed a restart/resync state-root mismatch, then reset the testnet and put all three VPSes on the exact same release artifact.

## Root Cause

- `v0.5.17` was published, signed, and installed on all VPSes, but the live chain diverged during restart/resync.
- US/SEA rejected a committed block after a burst of oracle attestation transactions.
- Accounts, stake, and moss roots matched, while contract storage roots differed.
- The missing scheduler conflict was native oracle opcode `30`: multiple oracle attestation transactions could update native oracle consensus state in parallel before mirroring into contract storage.
