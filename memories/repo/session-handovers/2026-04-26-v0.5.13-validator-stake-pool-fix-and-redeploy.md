# 2026-04-26 - v0.5.13 Validator Stake-Pool Fix And Clean-Slate Redeploy

## Summary

- Root cause of the slot-285 SEA failure was a non-consensus background stake-pool persistence task in `validator/src/main.rs`.
- The task could clone stale in-memory stake data, then overwrite RocksDB after a block had committed a newer stake pool. Because stake state participates in the state root, the next block could fail verification on the overwritten node.
- The fix removes the background stake-pool writer and leaves stake-pool persistence to deterministic block application, sync/genesis import, and explicit snapshot paths.
- Ghost validator cleanup is now enforced through stake-backed validator admission: announcements and block headers do not create validator-set entries, and startup drops unbacked persisted validator metadata.
