Exchange Integration

Native LICN exchange integration is in testnet-only readiness work until mainnet launch. Local and public testnet gates now cover metadata, operations policy, archive/history regressions, SDK/CLI boundaries, the three-validator exchange simulation, and the public faucet-backed testnet exchange simulation. The developer portal page is deployed and verified; the monitoring status page is approved for the current testnet-only package; exchange contact aliases are approved as security@lichen.network, exchange-ops@lichen.network, and business@lichen.network. The final testnet-only exchange package is published under exchange-testnet-v0.5.221.

Testnet package status
The current package is published for testnet-only integration under exchange-testnet-v0.5.221. Do not present it as mainnet-ready or accept mainnet LICN deposits until the mainnet launch exchange handoff passes, the readiness gate is rerun with full scope, and a signed mainnet-inclusive package is published.

Integration Anchors

Area Current source-backed rule
Package scope Testnet-only until mainnet launch. Mainnet exchange publication requires the launch runbook exchange handoff and full-scope public readiness.
Asset Native LICN uses 9 decimals; 1 LICN equals 1,000,000,000 spores.
Accounting Use raw spores as integers. RPC formatted LICN fields are display-only.
Address Native addresses match the Base58 prefilter and decode to exactly 32 bytes.
Native RPC Use canonical JSON-RPC on / with getSlot, getBlock, getTransaction, history, balance, and sendTransaction.
EVM compatibility Native listings use getNetworkInfo chain IDs; /evm eth_chainId is runtime compatibility data. Public testnet currently returns 0xca3f1595a6c25e9f.
Archive Exchange operations require archive-backed endpoints. State-only RPC is not enough.
Rollback Current rollback anchor is v0.5.221 until a newer signed rollback point is recorded.

Validation Gates

  • Keep address validation vectors passing before publishing or changing a regex.
  • Keep deterministic finality plus the selected exchange buffer validated on a local three-validator testnet.
  • Keep archive-backed getTransaction, getBlock, and address history covered across hot/cold data and restart.
  • Keep the deposit, credit, sweep, withdrawal, retry, and reconciliation simulation passing locally.
  • Clean up every local stack run and prove validator, custody, faucet, and source-chain mock processes are stopped.
  • Keep the public testnet exchange simulation evidence attached to the signed recovery release.
  • Keep the published exchange package tag, public developer page, and testnet-scope readiness gate aligned before outreach.
  • Do not publish a mainnet package until the mainnet launch exchange handoff and full-scope readiness gate pass.
