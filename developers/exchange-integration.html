Exchange Integration

Native LICN exchange integration is in readiness work. The local source-backed gates now cover metadata, operations policy, archive/history regressions, SDK/CLI boundaries, and the three-validator exchange simulation. Public endpoint readiness and operator approvals remain release blockers.

Draft status
This page is not an external listing package. Serious exchange outreach is blocked until public RPC/WebSocket readiness, public developer page deployment, status-page approval, incident aliases, final package tag selection, and public testnet simulation all pass.

Integration Anchors

Area Current source-backed rule
Asset Native LICN uses 9 decimals; 1 LICN equals 1,000,000,000 spores.
Accounting Use raw spores as integers. RPC formatted LICN fields are display-only.
Address Native addresses match the Base58 prefilter and decode to exactly 32 bytes.
Native RPC Use canonical JSON-RPC on / with getSlot, getBlock, getTransaction, history, balance, and sendTransaction.
Archive Exchange operations require archive-backed endpoints. State-only RPC is not enough.
Rollback Current rollback anchor is v0.5.215 until a newer signed rollback point is recorded.

Validation Gates

  • Keep address validation vectors passing before publishing or changing a regex.
  • Keep deterministic finality plus the selected exchange buffer validated on a local three-validator testnet.
  • Keep archive-backed getTransaction, getBlock, and address history covered across hot/cold data and restart.
  • Keep the deposit, credit, sweep, withdrawal, retry, and reconciliation simulation passing locally.
  • Clean up every local stack run and prove validator, custody, faucet, and source-chain mock processes are stopped.
  • Do not release until the public readiness gate and public testnet exchange simulation pass.

Next Step

The next readiness task is public endpoint recovery: testnet RPC must stop reporting stale readiness, mainnet RPC and WebSocket must stop returning Cloudflare 525, and this page must be deployed publicly with exchange-specific content before public testnet exchange validation can run.
